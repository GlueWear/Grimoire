# GRIMOIRE - Project Outline

## Overview
GRIMOIRE is a cyberpunk-styled collaborative chat web app inspired by Urbit. It features real-time messaging, hierarchical notes, cryptocurrency wallet integration, and AI agent webhooks.

**Repository:** https://github.com/GlueWear/Grimoire
**Stack:** Node.js + Express + Socket.IO (backend), Vanilla JS + HTML/CSS (frontend)

## Major Features

### 1. Authentication System
- **Urbit ID Login** - Connect MetaMask ظْ query Azimuth contract on Ethereum mainnet ظْ select owned planet ظْ login as `~planet-name`
- **Visitor Login** - Simple username with `@` prefix (e.g., `@visitor`)
- **Agent Login** - Bot accounts with `%` prefix (e.g., `%agent`)
- **Session Storage** - Uses `sessionStorage` (per-tab, not shared across tabs)
- **Uniqueness Enforcement** - Server rejects duplicate usernames

### 2. Notes System
- **Hierarchical Notes** - Notes can have children (nested notebooks)
- **ARS NOTORIA** - Special public "cover" note visible to all users
- **DMs** - Private notes between two users (`type: 'dm'`)
- **Note Sharing** - Invite users to private notes
- **Notifications** - Green dot system for unread messages, propagates to parent

### 3. Messaging
- **Real-time Chat** - Socket.IO-based instant messaging
- **Threaded Replies** - Reply to specific messages
- **@Mentions** - Tag users or bots
- **Artifacts** - Embedded content blocks (code, markdown, etc.)

### 4. Wallet System ($NOCK)
- **Wrapped NOCK on Base** - Contract: `0x9B5E262cF9bb04869ab40b19AF91D2dc85761722`
- **MetaMask Integration** - Real ERC-20 transfers via MetaMask
- **Username Resolution** - Send to `~username` ظْ server looks up ETH address
- **Transaction History** - Server-side tracking of sends/receives
- **Reply Sends** - Send NOCK as reply to a message
- **DM Sends** - Send NOCK to user ظْ creates/finds DM ظْ posts message

### 5. Agent/Bot System
- **Webhook Registration** - Bots register with name + webhook URL
- **@Mention Triggers** - Mentioning bot calls webhook with message
- **Bot Responses** - Can reply with text, create artifacts

### 6. Light Bike Game (WIP)
- **Turn-based Grid Game** - Two players on grid
- **Bot Integration** - Bots can play via webhooks
- **Artifact Display** - Game state rendered as artifact

## Server Architecture

### State Management (In-Memory)
```javascript
const state = {
  users: new Map(),        // socketId ظْ { name, type, id, currentNote }
  notes: {},               // noteId ظْ { id, name, type, creator, users, messages, threads, children, parent, ... }
  artifacts: {},           // artifactId ظْ { id, noteId, name, content, ... }
  wallets: {},             // username ظْ { balance, transactions }
  walletAddresses: {},     // username.toLowerCase() ظْ ETH address
  bots: {},                // socketId ظْ { name, webhookUrl, owner }
  takenUsernames: new Set(),
  pendingInvites: {}       // recipientName ظْ [{ noteId, inviter, ... }]
}
```

### Key Socket Events

**Authentication:**
- `join` ظْ `joinConfirmed` / `joinRejected`

**Notes:**
- `createNote` ظْ `noteCreated`
- `switchNote` ظْ receive note messages
- `shareNoteWithUser` ظْ `noteShared`
- `inviteToNote` ظْ `inviteReceived` / `noteInviteConfirmed`

**Messaging:**
- `message` ظْ broadcasts to note participants
- `createArtifact` ظْ `artifactCreated`

**Wallet:**
- `registerWalletAddress` - Store ETH address for user
- `nockSendConfirmed` - Log transaction after MetaMask confirms
- `getWallet` ظْ `walletUpdate`

**Bots:**
- `registerBot` - Register webhook URL
- Webhook called on @mention with `{ message, botName }`

## Development Setup

```bash
cd Grimoire
npm install
node server.js
# Open http://localhost:3000
```

### Testing Multi-User
Open multiple browser tabs - each tab has independent session (sessionStorage).

### Testing with Bots
Run separate bot server (e.g., ollama-bridge) that:
1. Connects to GRIMOIRE socket
2. Emits `registerBot` with webhook URL
3. Receives POST to webhook when @mentioned
4. Returns `{ reply: "..." }` to respond
